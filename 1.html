# Making a folder and a README file in Linux.

> mkdir Project

# mkdir creates a folder. Its first argument gives the folder a name, and is required.

# Click anywhere on this comment to continue.

> cd Project

# cd stands for "change directory". Folders are generally equivalents of directories. The first argument is the directory that it changes to. To change the directory to a parent, use "..". If you want to describe the children folder of the folder to go to, use /'s (e.g. cd folder/childfolder).

# Click anywhere on this comment to continue.

> touch README.txt

# The touch command creates a file. Its argument determines the name and the filetype of the resulting file.

# Click anywhere on this comment to continue.

> echo "This is a readme file." >> README.txt

# The echo command writes the first argument into something. It usually writes into the terminal. However, if arrows are used, it will write down the first argument into the file the arrows are pointing to. If the file referred to by the arrows doesn't exist, then it will be made with the first argument written into it.

# Click anywhere on this comment to continue.

> cat README.md

This is a readme file.

# cat writes the contents of the file referred to in the first argument right into the terminal. This concludes our first tutorial.
